# 关键词 · 产品属性匹配规则

> 标书产品分析的核心：**用检索词 + 扩展属性词**，在**详情正文**与**附件文本**中定位产品清单行，再抽取金额与参数。

## 1. 匹配画像 `MatchProfile`

| 字段 | 说明 |
|------|------|
| `keyword` | 用户检索词（如「工业相机」） |
| `primary_terms` | 主词拆分（逗号/空格/顿号） |
| `attribute_terms` | 同义词、品类词、常见参数词 |
| `model_patterns` | 型号正则（字母数字组合） |

## 2. 属性扩展（内置 + 可扩展）

检索词命中 `ATTRIBUTE_EXPANSIONS` 时自动追加属性词，例如：

- **工业相机** → 机器视觉、CCD、CMOS、面阵、线扫、像素、分辨率
- **光谱仪** → 光谱、波长、分辨率、nm
- **投影仪/投影** → 投影机、DLP、流明、亮度

未命中词典时：主词 + 主词拆字片段（≥2 字）作为弱匹配。

## 3. 文本来源与优先级

| 来源 | `参数来源` 标记 | 优先级 |
|------|----------------|--------|
| 附件 PDF/Word 解析正文 | `attachment:文件名` | 最高（字段最全） |
| 详情页正文产品表 | `detail_page` | 中 |
| 附件文件名/链接名 | `attachment_meta` | 低（仅线索） |
| 项目标题兜底 | `project_only` | 最低 |

合并：同一项目 `(产品名称, 型号)` 相似则合并，**附件优先覆盖**详情空字段。

## 4. 行级匹配打分

对每一候选产品行（或上下文块）计算 `match_score`（0–100）：

- 主词完整命中：+40
- 属性词命中（每个）：+8（上限 32）
- 行内含数量/单价/金额数字结构：+15
- 含规格参数键（分辨率/量程/精度/品牌）：+10

**入选阈值**：`match_score >= 35` 或该项目无更高分行时保留 Top3。

## 5. 参数抽取（L3 键值）

在匹配行 **前后 5 行** 或同行文本中抽取：

| 参数名 | 正则线索 |
|--------|----------|
| 品牌 | 品牌[:：]、制造商、厂家 |
| 型号 | 型号[:：]、Model、规格型号 |
| 分辨率 | 分辨率、万像素、× |
| 量程 | 量程、测量范围 |
| 精度 | 精度、准确度 |
| 数量 | `\d+` + 台/套/个/件 |
| 单价/金额 | 单价、总价、成交金额、万元 |

## 6. 数量 / 单价 / 总价：代码 vs AI

| 场景 | 推荐方式 | 说明 |
|------|----------|------|
| 政府采购「主要标的」制表符表（品目\\t标的\\t数量\\t单价\\t总价） | **代码** | `amount_fields.py`：表头识列 + `数量×单价≈总价` 校验，可纠正单价/总价列颠倒 |
| 数量带单位 `4.00(台)`、金额带逗号/万元 | **代码** | 规则解析 + 推算缺失列 |
| Word/PDF 合并单元格、列顺序混乱、无表头 | **代码优先** | 表头检测 + 行尾三数字启发；置信度 `low` 记入 `规格参数.金额置信度` |
| 扫描件 OCR 乱码、非表格散文清单 | **AI 兜底** | 仅对 `金额置信度=low` 的行调用 LLM（可选，控制成本） |

**结论**：约 **70–80%** 项目用增强代码即可；不必默认全量 AI。全量 LLM 慢、贵且仍可能幻觉。
