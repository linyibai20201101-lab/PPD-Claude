# 标书产品信息分析 · 统一数据模型

> 项目统计 + 产品行 + 附件参数 **统筹到同一套结构**，便于 Excel 导出与报告生成。

## 1. 项目层 `ProjectRecord`（L1）

一条中标/成交记录，来源：列表 CSV 或 `tender-info` job。

| 字段 | 说明 | 来源优先级 |
|------|------|------------|
| project_id | 内部 ID | 系统 |
| 项目名称 | | 列表 / 详情 |
| 发布时间 | | 列表 |
| 地区 | | 列表 |
| 预算金额_万元 | 项目金额 | 列表 → 详情正文 → 附件 |
| 项目类型 | 成交/招标等 | 列表 |
| 行业类型 | | 列表 / 规则识别 |
| 采购单位 | | 列表 / 详情 |
| 中标单位 | | 列表 / 详情 |
| 有附件 | bool | 列表 |
| 详情链接 | 剑鱼 URL | 列表 |
| 原文链接 | 源站 URL | 详情页「原文」按钮 |
| 附件状态 | pending / downloaded / parsed / failed | 系统 |
| 数据来源 | 剑鱼标讯 | 固定 |

**项目级统计（聚合用）**：对 L1 按地区、行业、中标单位、月份汇总 `预算金额_万元`、项目数。

## 2. 产品行层 `ProductLine`（L2）

一条记录 = 一个项目中的一行产品/物资明细。

| 字段 | 说明 | 来源优先级 |
|------|------|------------|
| line_id | | 系统 |
| project_id | 外键 | |
| 产品名称 | | 详情表 / 附件表 |
| 品牌 | | 附件 |
| 型号 | | 详情 / 附件 |
| 规格摘要 | 短文本 | 详情 / 附件 |
| 数量 | 数值 | 详情 / 附件 |
| 单位 | 台/套/个 | 附件 |
| 单价_元 | | 附件 |
| 行金额_元 | 数量×单价 或表内总价 | 附件 |
| 行金额_万元 | | 计算 |
| 参数来源 | detail_page / attachment:文件名 | 系统 |
| 匹配关键词 | 是否含检索词 | 规则 |

**产品级统计**：按 `产品名称`、`型号` GROUP BY → `出现次数`、`数量合计`、`金额合计`、`平均单价`。

## 3. 参数层 `ProductParam`（L3）

从附件/长文中拆出的键值对，一行一个参数。

| 字段 | 说明 |
|------|------|
| param_id | |
| line_id | 可空；仅项目级时挂 project_id |
| project_id | |
| 参数名 | 如 分辨率、测量范围 |
| 参数值 | |
| 原文摘录 | ≤200 字 |
| 页码或段落 | PDF 页 / Word 段 |

**参数统计**：某参数名下的取值分布（如分辨率 200万/400万 占比）。

## 5. 统筹导出 `products_master`（Excel 宽表）

每行 = **一个 ProductLine**，左侧冗余 L1 项目字段，右侧展开常用参数列（透视或固定列）：

```
项目名称 | 发布时间 | 地区 | 预算金额_万元 | 采购单位 | 中标单位 |
产品名称 | 品牌 | 型号 | 数量 | 单位 | 单价_元 | 行金额_万元 |
分辨率 | 量程 | 精度 | ... | 参数来源 | 附件文件名
```

## 7. 合并规则（去重）

- 同一 `project_id`：L1 以列表为准，详情/附件**补全**空字段，不覆盖已有非空金额除非附件更完整
- 同一项目下产品行：`(产品名称, 型号)` 相似度 > 阈值则合并数量/金额
- 详情页已有产品行 + 附件表重复：优先 **attachment** 字段，详情作补充

## 8. 与 tender-analysis 的差异

| 维度 | tender-analysis | 本模块 |
|------|-----------------|--------|
| 主键 | 项目 | 产品行 |
| 附件 | 一般不解析 | **必须链路** |
| 报告 | 15 项市场图表 | 产品总表 + 参数 TOP + 可选 LLM |
| 统计 | 区域/行业大盘 | 产品/型号/金额/数量统筹 |
