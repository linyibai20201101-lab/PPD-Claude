**评审日期**：2026-06-02  
**评审人角色**：项目经理 + 智能体治理  
**对照需求**：按「检索关键词 + 关键词产品属性」匹配标书正文与附件中的产品清单，再抓取金额与参数。

---

## 一、需求理解（对不对）

| 维度 | 结论 | 说明 |
|------|------|------|
| 方向 | **正确** | 与 `unified_schema.md` L2/L3 设计一致：以**产品行**为主键，而非仅项目级金额 |
| 匹配逻辑 | **正确** | 仅用项目标题或硬编码型号（如 TS-9230）会导致漏抓、误抓；必须 **keyword → 属性扩展 → 正文/附件清单行** |
| 数据来源 | **正确** | 详情正文 + 附件（招标文件）双通道；附件字段更全，应 **附件优先合并** |
| 产出 | **正确** | 金额、数量、型号、分辨率/量程/精度等参数，汇总到 `products_master.xlsx` + 统计 JSON |

**结论：业务路线与建设优先级判断正确，应作为本模块 P0 核心算法，而非可选优化。**

## 三、风险与偏离

| 风险 | 等级 | 缓解 |
|------|------|------|
| 检索词过宽/过窄 | 中 | 门户必填或默认带出 tender-info 关键词；报告展示 `匹配得分` 与 `匹配属性词` |
| 正文无表格仅叙述 | 中 | 降低阈值 + 上下文块抽取；附件 P1 补齐 |
| 剑鱼详情非全文 | 中 | P1 跟跳原文 + 下载附件 |
| 与 tender-analysis 重复 | 低 | 边界已文档化：本模块主表=产品行，不做 15 项市场 HTML |

**偏离说明**：无偏离智能体路线；属 L1 工具能力加深，符合「先 Tools 后 Agent」。

## 四、验收建议（项目经理）

1. 用真实任务 ID（如 `25fe31ddf9f4`）+ 明确关键词（如「工业相机」）跑一轮 **抓取详情**。
2. 打开 `products_master.xlsx`，检查：
   - `匹配关键词` / `匹配得分` 是否有值；
   - `产品名称` 是否与检索品类相关（非无关设备）；
   - `规格摘要` 是否含品牌/型号/分辨率等。
3. 对比 **仅列表 CSV** vs **详情后**：产品行数、金额字段补全率应上升。
4. P1 完成后复测：**有附件** 项目参数列完整度目标 ≥ **85%**。
